1 #1 Base

In base are severall classes and components defined which are required by other components.

1.1 #1.1 Color

Like in Google's material ui, colors are grouped in palettes by hue. Each contains 9 different shades, ranging from 100 (lightest) over 500 (base color) up to 900 (darkest).

Some special colors as white, black and transparent are included too, even though they don't have palettes on their own.

1.1.1 #1.1.1 Foreground

Foreground color classes are named as .sy-text-{color}-{shade} and just set the required color as the element's color, therefore the text is getting colorized.

Example

Blue text.

Red text.

Gray text.

Markup 
<p class="sy-text-blue-500">Blue text.</p>
<p class="sy-text-red-500">Red text.</p>
<p class="sy-text-gray-500">Gray text.</p>

1.1.2 #1.1.2 Background

Background color classes are named as .sy-{color}-{shade} and just set the required color as the element's background color.

Example

Blue background.

Red background.

Gray background.

Markup 
<p class="sy-blue-500 sy-text-white">Blue background.</p>
<p class="sy-red-500 sy-text-white">Red background.</p>
<p class="sy-gray-500 sy-text-white">Gray background.</p>

1.2.1 #1.2.1 Alignment

Text elements can be aligned with the following classes:

  • .sy-text-start: Text is aligned to the left (default) or right (rtl).
  • .sy-text-center: Text is aligned to the center.
  • .sy-text-end: Text is aligned to the right (default) or left (rtl).
  • .sy-text-justify: Text fills all available space.
  • .sy-text-justify-all: Like .sy-text-justify, but the last line is justified too.
  • .sy-text-match-parent: Matches the alignment of its parent.
  • .sy-rtl: Text flows from right to left.
  • .sy-ltr: Text flows from left to right (default).

1.3 #1.3 Layout

Like Bootstrap Syndesi-UI uses standartized classes for layout propertys like margin, padding and element sizes.

The code was adapted from the Bootstrap project.

The used directions are as follows:

  • t for top
  • r for right
  • b for bottom
  • l for left
  • x for x-axis (both left and right)
  • y for y-axis (both top and bottom)
  • (nothing) for all 4 directions

1.3.1 #1.3.1 Margin

Margin classes are defined as .sy-m{direction}-{size}. The size ranges from 0 (smallest) up to 5 (biggest).

Example

sy-mx-0 sy-mb-5

sy-mx-5

sy-mx-0 sy-mt-5

Markup 
<p class="sy-mx-0 sy-mb-5 sy-blue-500 sy-text-mono">sy-mx-0 sy-mb-5</p>
<p class="sy-mx-5 sy-blue-500 sy-text-mono">sy-mx-5</p>
<p class="sy-mx-0 sy-mt-5 sy-blue-500 sy-text-mono">sy-mx-0 sy-mt-5</p>

1.3.2 #1.3.2 Padding

Padding classes are defined as .sy-p{direction}-{size}. The size ranges from 0 (smallest) up to 5 (biggest).

Example

sy-px-0 sy-pb-3

sy-px-5

sy-px-0 sy-pt-3

Markup 
<p class="sy-px-0 sy-pb-3 sy-blue-500 sy-text-mono">sy-px-0 sy-pb-3</p>
<p class="sy-px-5 sy-blue-500 sy-text-mono">sy-px-5</p>
<p class="sy-px-0 sy-pt-3 sy-blue-500 sy-text-mono">sy-px-0 sy-pt-3</p>

1.4 #1.4 Grid

The grid component provides several classes to build responsive layouts from the ground up.

1.4.1 #1.4.1 Container

The container is the main wrapper for the content and is centered. It accepts the optional class .sy-gap-{size} with sizes from 0 to 5 which changes its padding.

May be removed in the future (no additional functionality over .sy-px-{size})

Example

Content

Markup 
<div class="sy-container sy-gap-3 sy-blue-500">
  <p class="sy-blue-400">Content</p>
</div>

1.4.2 #1.4.2 Row

The row fills the whole horizontal space and can be splitted by columns. Like the container, rows can accept the optional class .sy-gap-{size} in order to provide different gap sizes.

Example

Column A

Column B

Column C

Markup 
<div class="sy-row">
  <div class="sy-col">
    <p class="sy-blue-500">Column A</p>
  </div>
  <div class="sy-col">
    <p class="sy-blue-500">Column B</p>
  </div>
  <div class="sy-col">
    <p class="sy-blue-500">Column C</p>
  </div>
</div>

1.4.3 #1.4.3 Column

In contrast to the row can the column be filled with content and is responsible for its own width. The class is defined as .sy-col-{width}, whereas the width ranges from 1 (1/12 = 8.3%) to 12 (12/12 = 100%). When no width is defined (e.g. .sy-col), the column will use as much space as possible and can therefore be used for divisions with the same width, e.g. 5 columns with equal width in a single row.
